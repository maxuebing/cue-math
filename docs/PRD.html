01产品概述

1.1 项目背景

中式八球实战中，当目标球被障碍球挡住直线与单库路径时，玩家必须使用2 库解球——母球经两条库边反弹后击中目标球，要求玩家快速进行"镜像展开"心算。目前缺乏线上专项训练工具反复练习这一核心技能。

1.2 产品定位

CueMath 是一款面向中式八球玩家的 H5 训练应用，通过"多障碍球挡路 → 自由选库 → 镜像法路径计算 → 即时轨迹反馈"的训练循环，帮助玩家通过碎片化时间掌握 2 库解球的心算技巧。

1.3 业务目标

填补训练空白

差异化

提供线上"2 库 kick 专项训练"，涵盖多障碍球逼迫、自由选库、镜像法验证。

提升实战精度

价值

错题本 + 盲区热力图，帮助玩家识别"哪条库/哪个象限"薄弱点，针对性补强。

多样化训练

深度

新手（固定母球）→ 进阶（随机）→ 大师（错 3 次结束）→ 限时（60s ± 时间）。

02目标用户与使用场景

2.1 用户画像

P1 · 中八进阶玩家

核心

中式八球进阶玩家，需要掌握 2 库解球的镜像法心算技巧。痛点：障碍球挡路时不知道该撞哪两条库、撞点算不准，缺乏线上反复练习工具。

P2 · 中八新手

潜力

中式八球初学者，需要从固定母球的简单场景入门，学习 2 库解球的基本思路。痛点：镜像法概念抽象，缺乏交互式教学工具。

P3 · 竞技挑战者

高频

追求极限解球速度与连击成就感，需要限时压榨和高难度挑战。痛点：现有工具无倒计时 / 错题限次 / 难度梯度。

2.2 用户痛点汇总

  • 脱离实体球桌无法训练走位心算——线下场地与时间成本高。
  • 公式抽象难以理解——缺乏"输入公式 → 看见轨迹"的直观反馈。
  • 不知算力盲区在哪——缺乏错题数据可视化与针对性复盘。

2.3 典型使用场景

通勤 / 排队等碎片时间（5–10 分钟）：刷一组限时题保持心算手感；
球局开始前热身：提前激活颗星公式心算反应；
赛后复盘：针对错题区间进行专项训练。

03核心功能需求

3.1 核心状态机（Game Loop）

单局训练由以下五个状态顺次流转，构成 Core Loop：

图 3-1　核心状态机：Init → Generate → Wait_Input → Animation → Settle，Animation 处分命中 / 错误两支

状态说明

  • Init：加载球桌素材、音效，初始化刻度系。
  • Generate：根据当前模式（新手 / 进阶 / 大师）从题库策略中生成一组 (A, B) 坐标，渲染母球与目标球。
  • Wait_Input：开启库边热区监听，等待用户点击第一库目标点 C_user。
  • Animation：锁死交互。计算正确点 C_real。
    • 若 C_user == C_real，播放命中动画及特效。
    • 若 C_user != C_real，播放错误路线动画，随后展示正确轨迹与公式面板。
  • Settle：更新得分、连击数（Combo）、记录耗时与错题，进入下一题（返回 Generate）。

3.3 镜像法算法（反射定律展开）

2 库解球的核心算法基于镜像法（Reflection Method）：目标球关于第二库镜像，再关于第一库镜像，母球到最终虚像的直线交第一库得 P1，P1 到一阶虚像的直线交第二库得 P2。

T1 = mirror(Target, c2) → T2 = mirror(T1, c1) → P1 = line(Cue, T2) ∩ c1 → P2 = line(P1, T1) ∩ c2

算法要点：

  • 双库镜像展开：两次镜像等价于母球直线瞄准虚像，交点即为两库撞点（反射定律保证）。
  • 多障碍球定向放置：题面随机放 2-3 个障碍球，定向挡住直线与全部 4 条 1 库路径，迫使必须 2 库解球。
  • 自由选库：4 条库边均可点击选择第一库撞点，不限高亮特定库，考验玩家的库边判断力。
  • 内缩边界校验：撞点坐标必须在球桌库边段内（内缩球半径），防止路径跑出球桌外。

算法策略采用几何映射（伪物理），刻意不引入物理引擎——镜像法给出精确的反射定律路径，保证轨迹可预测、可复现。

07非功能需求

⚡ 性能

动画渲染 60fps（PixiJS WebGL）；首屏加载 ≤ 2s（4G 网络）；单题动画响应 ≤ 100ms。

📱 兼容性

iOS Safari、Android Chrome、微信 webview；适配 360px–414px 主流移动宽度；横屏时提示锁定竖屏。

🔒 安全

本地数据不上传，保护隐私；限时模式做前端时间篡改校验；Phase 3 排行榜需服务端二次校验防作弊。

♿ 可访问性

刻度数字对比度 ≥ AA；支持系统字体放大；关键操作提供非颜色反馈（震动 / 音效）。

🧩 可维护性

状态机与算法模块化，便于扩展新模式；题库策略与渲染解耦，算法层可独立测试。

📦 体积

H5 首屏资源（gzip）目标 ≤ 500KB，PixiJS 按需加载，素材采用 SVG / WebP 压缩。

A.1 术语表

台球与颗星公式相关术语
术语释义
颗星（Diamond）球桌库边的刻度标记点，是颗星公式的计算基准。
母球（Cue Ball）被球杆击打的白色球，公式中记作 A（出发点）。
目标球（Object Ball）需要被击中的球，公式中记作 B（目标点）。
库（Cushion / Rail）球桌的边框；"一颗星"= 撞一次库。
三库（Three-cushion）母球需撞击库边三次再命中目标球的玩法。
颗星公式（Diamond System）利用颗星刻度进行走位计算的几何公式：A − B = C。
Combo（连击）连续答对的次数，用作得分加成与成就感反馈。
