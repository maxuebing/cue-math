# 《台球颗星公式训练》H5 应用设计说明书

## 1. 项目概述
*   **项目背景**：台球（尤其是三库台球与九球）实战中，玩家需熟练应用“颗星公式（Diamond System）”进行走位计算。目前缺乏脱离实体的线上专项脑力训练工具。
*   **业务目标**：打造一款高流畅度、强反馈的 H5 训练应用，帮助玩家通过碎片化时间形成心算肌肉记忆。
*   **目标平台**：移动端 H5 / 微信小程序。

## 2. 系统架构与技术实现
*   **前端框架**：Vue 3 / React（用于 UI 状态管理、计分板、弹窗交互）。
*   **核心画布渲染**：PixiJS（专注于 2D 桌面绘制、球体动画渲染）。
*   **轨迹算法策略**：**基于几何映射的伪物理计算**。
    *   *状态分离*：不引入 Box2D 等物理引擎。球桌视为 2D 坐标系。
    *   *路径生成*：预先建立颗星刻度到二维坐标的映射字典。根据公式 `A (出发点) - B (目标点) = C (第一库瞄准点)`，计算出多段折线的端点坐标 `(x1, y1) -> (x2, y2) -> (x3, y3)`。
    *   *动画执行*：使用 Tween 动画库控制母球沿生成的折线序列按匀速/线性减速平移。

## 3. 功能模块设计

### 3.1 核心状态机（Game Loop）
*   `Init`：加载球桌素材、音效，初始化刻度系。
*   `Generate`：根据当前模式（新手/进阶/大师）从题库策略中生成一组 `(A, B)` 坐标，渲染母球与目标球。
*   `Wait_Input`：开启库边热区监听，等待用户点击第一库目标点 `C_user`。
*   `Animation`：锁死交互。计算正确点 `C_real`。
    *   若 `C_user == C_real`，播放命中动画及特效。
    *   若 `C_user != C_real`，播放错误路线动画，随后展示正确轨迹与公式面板。
*   `Settle`：更新得分、连击数（Combo）、记录耗时与错题，进入下一题（返回 Generate）。

### 3.2 模式与关卡规则
| 模式 | 生成逻辑 | 通关/失败条件 | 核心目的 |
| :--- | :--- | :--- | :--- |
| **新手模式** | 母球固定 (50), 目标球随机 | 连续答对 10 题晋级 | 熟悉基本公式与交互 |
| **进阶模式** | 母球随机, 目标球随机 | 连续答对 15 题晋级 | 建立动态寻点计算能力 |
| **大师模式** | 引入短边/异常角度修正逻辑 | 错 3 题即结束 | 攻克公式死角与修正值 |
| **限时模式** | 全随机，初始 60 秒 | 答对+2s，答错-5s，倒计时归零结束 | 极限压榨心算速度 |

### 3.3 数据统计与存储
*   **本地存储（Local Storage）**：记录用户最高分、各模式解锁状态。
*   **错题矩阵**：后台维护一个二维矩阵 `ErrorMatrix[出发点区间][目标点区间]`，每次答错时对应权重 +1。训练结束后，根据矩阵高亮权重区，生成“算力盲区报告”。

## 4. 交互与 UI 规范
*   **视图布局**：竖屏模式。上半屏 60% 为二维球桌区域（视角自上而下），下半屏 40% 为控制台（得分、计时器、辅助选项、公式解析板）。
*   **交互热区**：方案采用“库边直选”。第一库对应的颗星点（0-50）设置不可见的点击热区（宽度需适配手指点击，防止误触）。
*   **UI 元素**：
    *   **球桌**：极简扁平风（Flat Design）或拟物绿呢台布，刻度数字字号需保证移动端清晰可读。
    *   **轨迹线**：正确路线（绿色，实线，透明度 0.8），错误路线（红色，虚线，透明度 0.6）。

## 5. 项目演进路线（Phase 规划）
*   **Phase 1（MVP 版本）**：实现进阶模式逻辑、核心坐标系映射算法、Tween 轨迹动画、基础库边点击交互。目标是跑通单局核心循环（Core Loop）。
*   **Phase 2（体验完善）**：加入新手/大师/限时模式，引入音效引擎，实现错题本数据统计算法，增加“线索盲抽”开关。
*   **Phase 3（社交与商业化）**：接入微信小程序生态，实现微信好友排行榜（基于限时模式得分），一键生成“台球脑力测试报告”海报用于社交裂变。
